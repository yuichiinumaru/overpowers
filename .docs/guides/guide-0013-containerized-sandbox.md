# Guide 0013: Containerized Sandbox

## Overview
The **Overpowers Sandbox** is an isolated Docker execution environment designed for running tests, executing agent-written scripts, and performing deep research without affecting your host system.

## Key Features
- **Isolation**: Prevents accidental file deletion or system corruption on the host.
- **Reproducibility**: Pre-configured with Python 3.11, Node.js 20, and all necessary system dependencies (including Playwright).
- **Seamless Integration**: Automatically maps your host user (UID/GID) and mounts your project root into the container.

## Usage

### 1. Launching the Sandbox
Use the launcher script to start the container in the background:
```bash
./scripts/orchestration/sandbox-launcher.sh up
```

### 2. Executing Commands
You can run commands directly inside the sandbox:
```bash
./scripts/orchestration/sandbox-launcher.sh exec "python3 my_script.py"
```

### 3. Entering the Sandbox (SSH)
Connect to the sandbox for interactive work:
```bash
./scripts/orchestration/sandbox-launcher.sh ssh
```
*Note: Default password is `overpowers`.*

### 4. Stopping the Sandbox
When finished, stop the container to release resources:
```bash
./scripts/orchestration/sandbox-launcher.sh down
```

## When to use the Sandbox
- **Dangerous Operations**: Any task involving `rm`, complex regex replacements, or system configuration changes.
- **Untrusted Code**: Running code generated by LLMs that hasn't been manually reviewed.
- **Dependency Isolation**: When you need a specific tool version that conflicts with your host.
- **Web Scraping/Testing**: Using Playwright/Puppeteer in a clean, consistent environment.

## Customization
You can customize the sandbox by editing `sandbox/Dockerfile`. After making changes, rebuild the image:
```bash
./scripts/orchestration/sandbox-launcher.sh build
```
