   * Agente: Rust Engineer (agents/ovp-rust_engineer.md)
       * Por que é bom: É um perfil técnico profundo que inclui
         checklists rigorosos para conformidade com clippy::pedantic,
         verificação com Miri (para blocos unsafe), proficiência no
         sistema de tipos (GATs, Pin, PhantomData) e estratégias de
         otimização zero-cost.
   * Agente: Rust Expert (agents/ovp-rust_expert.md)
       * Por que é bom: Focado em Rust idiomático. É ideal para garantir
         que o código utilize padrões de propriedade (ownership) e tempos
         de vida (lifetimes) de forma limpa, priorizando abstrações
         seguras e tratamento de erros com Result<T, E>.
   * Workflow: MASSOP-R3: Rust Rewrite Strategic Planning
     (workflows/ovp-20-rust-rewrite-plans.md)
       * Por que é bom: Um protocolo massivo de nível industrial para
         migração de sistemas legados para Rust. Ele foca em logística,
         "isomorfismo de estado" e planejamento de grafos de execução
         (DAG). É o workflow mais avançado para projetos Rust de grande
         escala.
   * Skill: ACG Rust Teacher (skills/coding/languages/teacher/SKILL.md)
       * Por que é bom: Embora tenha uma temática lúdica (ACG/Anime), é
         uma ferramenta excelente para explicar conceitos complexos de
         ownership e borrowing para outros agentes ou desenvolvedores,
         garantindo que a base conceitual do código esteja correta.

  🔍 2. Code Review (Mestria e Ferramentas)

  Recursos focados em identificar vulnerabilidades, gargalos de
  performance e dívida técnica.

   * Agente: Code Review Master (agents/ovp-code_review_master.md)
       * Por que é bom: Um revisor generalista de alto nível com foco
         triplo: Segurança (vulnerabilidades/injeções), Performance
         (algoritmos/N+1 queries) e Manutenibilidade. Inclui checklists
         detalhados e exemplos de padrões "Bom vs Ruim".
   * Workflow: Code Review Guide (workflows/ovp-code-review.md)
       * Por que é bom: Serve como a "bíblia" de revisão para o sistema.
         Cobre desde funcionalidade básica até acessibilidade e
         compatibilidade, definindo como o feedback deve ser estruturado
         (Blocker vs Suggestion).
   * Workflow: Write Tests (workflows/ovp-write-tests.md)
       * Por que é bom: Um processo orquestrado que utiliza agentes de
         revisão de cobertura para garantir que toda lógica crítica de
         negócio esteja testada. Essencial para projetos Rust onde a
         confiança nos testes é fundamental.
   * Skill: ast-grep (skills/coding/review/ast-grep/SKILL.md)
       * Por que é bom: Ferramenta escrita em Rust que realiza buscas
         estruturais e linting baseado em AST (Abstract Syntax Tree). É
         extremamente rápida e precisa para refatorações cirúrgicas e
         auditorias de segurança em Rust.
   * Skill: Code Analyzer (skills/coding/review/code-analyzer/SKILL.md)
       * Por que é bom: Suporta Rust nativamente e é especializado em
         análise de fluxo de dados e reconhecimento de padrões de DDD
         (Domain-Driven Design), como Agregados e Entidades.
