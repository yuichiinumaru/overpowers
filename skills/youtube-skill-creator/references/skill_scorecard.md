# Scorecard Universal para Criação de Skills

Use este scorecard para avaliar se um procedimento descrito em um vídeo é um bom candidato para se tornar uma skill. Atribua notas de 0 a 2 para cada critério (0=ruim, 1=médio, 2=ótimo).

## Critérios de Avaliação

1.  **Frequência/Volume:** A tarefa acontece com frequência (diária/semanal)?
2.  **Repetitividade:** É um trabalho mecânico (cliques, copiar-colar)?
3.  **Regra Clara:** Pode ser expresso como "se/então" com pouca ambiguidade?
4.  **Estabilidade:** O processo é maduro e muda pouco?
5.  **Entradas Padronizadas:** Os inputs são estruturados (APIs, planilhas, formulários)?
6.  **Complexidade:** Possui poucos passos e pontos de decisão?
7.  **Ganho de Precisão:** A automação reduziria erros humanos comuns?
8.  **Testabilidade:** É fácil criar casos de teste isolados?
9.  **Reuso/Portabilidade:** Aplica-se a vários contextos (não é um hack local)?
10. **Segurança/Risco:** O impacto de um erro é controlado ou possui guardrails?

## Sinais de Oportunidade no Vídeo

- **Workflow definido:** Início, meio e fim com critério de "concluído" claro.
- **Receita operacional:** O vídeo foca no "como fazer" (passo a passo).
- **Repetição de padrão:** O apresentador aplica o mesmo método em diferentes exemplos.
- **Checkpoints:** Existem validações explícitas ("confira X antes de Y").

## Decisão

- **Total >= 15:** Excelente candidato. Criar skill imediatamente.
- **Total 10-14:** Candidato médio. Avaliar se pode ser um update em skill existente.
- **Total < 10:** Baixo ROI. Apenas anotar a ideia se for muito inovadora.
