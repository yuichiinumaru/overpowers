# 小红书搜索与内容抓取 SOP

## 0. 默认行为

- 输入：`keyword`
- 默认样本量：`top_n=20`
- 输出格式：Markdown 列表（标准字段）

## 1. 搜索流程

1. 进入小红书搜索入口。
2. 输入关键词并执行搜索。
3. 在结果页按可见顺序选择帖子，优先进入帖子卡片主体。
4. 抓取到达 `top_n` 或结果耗尽后结束。

## 2. 抓取字段（标准字段）

每条记录必须包含：
- 标题
- 作者
- 链接
- 发布时间
- 点赞数
- 收藏数
- 评论数
- 摘要
- 关键词

字段缺失时返回空值并标注“页面未提供”，不要臆造。

## 3. Markdown 输出规范

输出结构建议：

```markdown
# 搜索抓取结果：<keyword>
- 样本量：<N>

## 1. <标题>
- 作者：...
- 链接：...
- 发布时间：...
- 点赞：...
- 收藏：...
- 评论：...
- 摘要：...
- 关键词：...
```

## 4. 质量控制

- 去重：同链接只保留一条。
- 可读：摘要保留核心信息，避免长段复制。
- 可复用：字段顺序固定，便于后续分析脚本解析。

## 5. 异常与降级

- 搜索结果为空：返回“无结果”并建议换词。
- 页面结构变化导致字段抓取不全：先保留已抓字段并标注缺失。
- 连续两次进入失败：停止并输出已抓取部分结果。
