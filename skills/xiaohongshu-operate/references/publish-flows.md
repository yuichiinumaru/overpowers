# 小红书发布链路（默认保存草稿）

## 0. 发布类型

- 普通图文（`post_type=image`）
- 长文（`post_type=longform`）
- 视频（`post_type=video`）

发布三要素（必填）：
1. 封面
2. 标题
3. 正文

## 1. 普通图文发布（默认优先，三段式仅在本节生效）

### 1.1 输入拆分（混合文本兜底）

当用户未显式提供 `image_text/title/body` 时，先做拆分：

1. 优先按标记拆分：`封面`/`图片文案`、`标题`、`正文`。
2. 无标记时：
   - 首行作为标题；
   - 其余作为正文；
   - 图片编辑文案（`image_text`）留空。

### 1.2 强制写入顺序与焦点校验

1. 进入发布页并选择图文发布入口。
2. 上传首图/多图素材（来自 `images[]`）。
3. 按顺序写入：
   - 图片编辑区写入 `image_text`（若为空则跳过）；
   - 标题区写入 `title`；
   - 正文区写入 `body`。
4. 每一步写入前确认焦点在目标输入框，写入后回读校验对应区域文本。
5. 若发现错位（例如正文进了标题框），清空错误输入框并按三段顺序重填。

### 1.3 图文超限处理（仅图文）

1. 以上传页 UI 的实时计数器/超限提示为准。
2. 标题或正文超限时，按规则压缩：
   - 去除重复标点与冗余空行；
   - 删除重复语义句；
   - 保留核心信息后截断到允许长度。
3. 压缩后重新写入并再次检查 UI 计数。
4. 若仍超限，返回最小改动建议并停在可编辑状态，不继续保存草稿。

### 1.4 草稿保存

1. 追加必要话题标签（如用户提供）。
2. 校验三要素完整且无超限。
3. 执行“保存草稿”。
4. 返回草稿保存结果与可继续编辑入口。

## 2. 长文发布

1. 进入长文创作入口。
2. 填写长文标题与正文结构（直接使用给定内容）。
3. 长文不应用图文三段式顺序和图文字数压缩规则。
4. 校验内容完整性与可见状态。
5. 执行“保存草稿”。
6. 回传草稿结果，等待后续修改或发布指令。

## 3. 视频发布

1. 进入视频发布入口。
2. 上传视频素材（`video_file`）。
3. 补齐封面、标题、正文。
4. 视频不应用图文三段式顺序和图文字数压缩规则。
5. 校验可见范围与基础设置。
6. 执行“保存草稿”。
7. 回传草稿结果，不自动正式发布。

## 4. 草稿完成判定（统一标准）

满足以下条件即视为“草稿完成”：
- 封面、标题、正文均已填写。
- 已执行保存草稿动作且界面有保存成功信号。
- 当前状态可继续编辑或后续发布。

## 5. 失败与降级

- 缺图/缺标题/缺正文：先返回缺口清单，等待补齐。
- 页面入口异常：切到稳健路径（重进发布页或改入口），最多重试一次。
- 保存失败连续两次：停止自动动作，回传当前页面状态与建议人工操作点。
