# 域账号登录引导

当检测到需要登录时（URL 包含 `login.alibaba-inc.com`），按以下步骤引导用户完成登录。

---

## 步骤 1: 切换到扫码登录

### 1.1 获取页面元素

使用 `mcp__browser-use__take_snapshot` 获取页面元素树。

### 1.2 定位扫码登录按钮

**页面结构说明**：
- 登录页面底部有"其它登录方式"分隔线 (`separator`)
- 分隔线下方有 4 个登录方式图标按钮 (`generic haspopup="menu"`)
- 按钮顺序（从左到右）：密码登录、可信设备验证登录、**扫码登录**、其他

**定位方式**：
1. 在 snapshot 中查找 `separator` 元素（包含"其它登录方式"文本）
2. 找到其后的第 **3** 个 `generic haspopup="menu"` 元素，即为扫码登录按钮
3. 或者：如果当前页面 `heading level="1"` 已显示"扫码登录"，则无需切换

### 1.3 点击切换

```
mcp__browser-use__click(uid=<扫码登录按钮的uid>)
```

### 1.4 验证切换成功

再次 `take_snapshot`，确认 `heading level="1"` 文本为"扫码登录"。

---

## 步骤 2: 截图二维码并发送给用户

1. 使用 `mcp__browser-use__take_screenshot` 截取当前页面
2. 将截图直接展示给用户（截图会自动包含二维码）
3. 向用户提示：

```
请使用阿里钉或阿里郎扫描上方二维码完成登录
```

---

## 步骤 3: 等待登录完成

**轮询检测登录状态**：

1. 每隔 5 秒使用 `mcp__browser-use__take_snapshot` 检查页面状态
2. 检查当前 URL 是否已离开 `login.alibaba-inc.com`
3. 最长等待 120 秒

**成功标识**：
- URL 不再包含 `login.alibaba-inc.com`
- 页面出现工单系统的元素（如"创建工单"按钮、工单列表等）

**超时处理**：
- 如果 120 秒内未完成登录，询问用户是否需要刷新二维码重试
- 如需重试，刷新页面后返回步骤 1

---

## 步骤 4: 记录登录账号并继续

登录成功后：

1. 使用浏览器工具导航到工单系统：`https://t.aliyun-inc.com/workbench/list`
2. 获取页面元素，从页面右上角用户信息区域提取**当前登录账号**
3. **记录账号信息**：将登录账号保存，用于后续填写工单的"处理人"字段
4. 确认页面正常加载后，返回主流程继续执行

**账号信息用途**：创建工单时，"处理人"字段自动填充为当前登录账号

---

## 元素定位参考

| 元素 | 特征 | 用途 |
|------|------|------|
| 页面标题 | `heading level="1"` | 判断当前登录方式 |
| 其它登录方式分隔线 | `separator` 包含"其它登录方式" | 定位登录方式按钮区域 |
| 扫码登录按钮 | 分隔线后第 3 个 `generic haspopup="menu"` | 切换到扫码登录 |
| 二维码区域 | 页面中部图片区域 | 截图展示给用户 |

---

## 常见问题处理

| 问题 | 处理方式 |
|------|----------|
| 二维码过期 | 刷新页面重新获取二维码 |
| 找不到扫码登录按钮 | 检查 snapshot 中 `haspopup="menu"` 元素，尝试逐个点击 |
| 登录后仍跳转到登录页 | 可能是权限问题，提示用户检查账号权限 |
| 页面加载异常 | 刷新页面重试，最多 3 次 |
