# 口语转书面语规则

> ⚠️ **重要提示**：本文件为通用转换规则参考。在使用时，需结合 `SKILL.md` 中的风格档案，优先选择**亲切自然**的转换选项，避免过于正式的书面语。

## 核心原则

1. **保留信息**：转换表达方式，不丢失核心内容
2. **保留风格**：书面化的同时保留作者个人风格（Antonia 风格偏亲切，慎用过于正式的选项）
3. **适度转换**：不必完全消除口语感，保持适当的亲和力

## 句式转换规则

### 冗余删除

**删除无意义的开头语**：
- ❌ "那个，我想说的是..."
- ✅ 直接陈述内容

**删除重复表达**：
- ❌ "其实本质上来说，根本的原因是..."
- ✅ "根本原因是..."

**删除口头填充**：
- ❌ "就是...就是那种..."
- ✅ 直接描述

### 碎片整合

将口语中的碎片化信息整合为完整句子：

**口语原文**：
> 那天我去了那个咖啡馆，就是我们上次去过的那个，然后点了杯咖啡，美式的那种，然后就坐在窗边。

**书面转换**：
> 那天，我去了我们曾一起造访的那家咖啡馆，点了一杯美式，坐在窗边。

### 逻辑显化

将口语中隐含的逻辑关系明确化：

**口语原文**：
> 他没来，我们就走了。

**书面转换**（根据上下文选择）：
- 因果：因为他没来，所以我们就离开了。
- 时间：他迟迟未到，我们便先行离去。
- 条件：既然他无法前来，我们决定先走。

## 结构重组规则

### 口语结构特点

口语通常是"想到哪说到哪"，表现为：
- 时间线混乱
- 插入语多
- 主次不分
- 重复啰嗦

### 书面重组策略

| 问题 | 处理方法 |
|------|----------|
| 时间线混乱 | 按时间顺序或逻辑顺序重排 |
| 插入语过多 | 将插入内容移至合适位置或脚注 |
| 主次不分 | 识别核心观点，突出主要内容 |
| 重复表达 | 合并同类信息，删除冗余 |

### 示例

**口语转录**：
> 我那天去面试，对了先说一下背景，就是我已经准备了很久了，大概两三个月吧，然后那天去了，发现面试官特别厉害，是个大佬，我就有点紧张，但是最后还是过了，就是后来他们给我发offer了，我特别开心。

**书面重组**：
> 经过两三个月的精心准备，我终于迎来了那场面试。面试官是业内资深人士，这让我不免有些紧张。但最终，我顺利通过了面试，并收到了offer，心中满是欣喜。

## 保留口语感的情况

以下情况可保留适度口语感：

| 场景 | 说明 |
|------|------|
| **作者风格本就口语化** | 遵循风格档案，保持一致性 |
| **对话/引语** | 引用他人说话保留原貌 |
| **情感高潮处** | 适当口语化可增强感染力 |
| **与读者互动** | "你有没有这种感觉？"可保留 |
| **个人标记** | 作者的标志性口语表达应保留 |

- [ ] 无意义的语气词已删除

- [ ] 逻辑关系已明确
- [ ] 结构已按逻辑重组
- [ ] 保留了作者的个人标记
- [ ] 整体语气与风格档案一致
- [ ] 核心信息未丢失
