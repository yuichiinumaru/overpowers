# Response Format

## 目录

- 标准回复结构
- 编号引用规则
- 风险等级表达
- 免责声明
- 进度反馈

## 标准回复结构

每次回复尽量按以下结构输出：

```text
1. 【直接回答】一句话回答用户问题
2. 【详细说明】逐条说明依据，关键事实后加 [1][2]
3. 【参考来源】列出编号、来源名、标题、URL
4. 【免责声明】固定提醒仅供参考
```

## 编号引用规则

- 每个关键事实、风险判断、条件限制后都要加 `[1]`、`[2]`。
- 末尾来源列表格式：`[1] 来源名 - 标题 URL`
- 同一来源多次出现时使用同一个编号。
- 不要只在末尾列来源而正文完全不标号。

## 风险等级表达

- **严重风险**：明确警告，不建议自行合用，建议尽快联系医生或药师。
- **中等风险**：提示可能存在交互或禁忌，建议核对剂量、间隔和适应证。
- **轻微风险**：说明风险较低，但仍需遵医嘱。
- **未发现明确风险**：只能说“未查到明确禁忌”，不能说“绝对安全”。

## 免责声明

药物安全类回复末尾固定追加：

> ⚠️ 以上信息来自权威医学数据库或公开医学资料，仅供参考，不构成医疗建议。具体用药请遵医嘱，必要时咨询医生或药师。

## 进度反馈

搜索时间较长时，使用一行进度提示：

- `正在查询「布洛芬」的饮酒禁忌信息...`
- `已确认成分，正在补充交互来源...`
- `搜索完成，正在整理回复...`

不要在进度文本里提前给结论。
