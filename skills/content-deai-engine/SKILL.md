# Content De-AI Engine

目标：把“信息正确但机器感重”的内容，重写成“有人味、可传播、可追溯”的平台稿。

## 0) 先决条件

开始写作前，必须先固定三项：
- 时间窗（例如：今日热点 / 本周观点）
- 素材来源（ainews/trading/macro/用户提供）
- 目标平台（小红书 / X / 知乎）

如果素材不足，必须明确写：
- 「今日暂无该方向素材」
- 或「基于昨日素材」
- 无法验证的信息标注「待核实」

## 1) AI 味诊断（先诊断，再重写）

先用 `references/anti-ai-patterns.md` 检查草稿，统计三类问题：
- 结构问题：三段式模板（首先/其次/最后）、机械连接词堆叠
- 语言问题：空泛大词、过度正确、缺少第一人称体验
- 传播问题：没立场、没细节、没行动建议

输出一行诊断结论：
- `AI 味等级: 高/中/低 + 主要原因（不超过 3 条）`

## 2) 四步重写法（必须按顺序）

1. **去模板**：删掉机械连接词、套话、教科书语气
2. **加细节**：至少补 2 个真实细节（时间/场景/数字/代价/踩坑）
3. **立观点**：给明确站位（支持/反对/条件成立）
4. **给动作**：结尾提供 1 个可执行动作或提问，促进互动

重写原则：
- 短句优先，允许停顿和口语化
- 不追求“完美书面语”，追求“像真人在说话”
- 不编造事实，不夸大结论

## 3) 平台适配

按 `references/platform-templates.md` 选择平台模板：
- 小红书：痛点场景 → 亲历细节 → 3步做法 → 互动提问
- X：反直觉观点 → 证据点 → 行动建议
- 知乎：问题边界 → 反例/证据 → 方法与适用条件

每次至少提供：
- 2-3 个标题候选（A/B）
- 1 个主稿
- 1 条评论区首评（用于补充信息或引导讨论）

## 4) 标准输出格式（强制）

每条内容必须包含：

- 目标受众
- 核心观点（一句话）
- 信息来源（ainews/trading/macro/用户输入）
- 核验状态（已核验/待核实）
- 平台适配说明（小红书/X/知乎）

输出顺序固定：
1. AI 味诊断
2. 标题候选
3. 正文
4. 评论区首评
5. 标签建议
6. 发布前风险提示

## 5) 质量门禁（发布前）

发布前执行 `references/preflight-checklist.md`。

最低门槛：
- 有明确目标受众
- 有明确观点，不中性空话
- 有可追溯来源
- 至少 2 处具体细节
- 无高频 AI 套话

如任何一项不满足，先修稿再给发布建议。

## 6) 三角色自检（单轮）

- 传播专家：标题与开头是否有点击动力
- 平台算法专家：格式和关键词是否符合平台分发习惯
- 魔鬼代言人：给出 1 条最强反对理由

若反对理由无法反驳，必须再改一轮。

## 7) 边界

- 不做未确认发布动作
- 不输出投资/医疗等高风险结论式建议
- 不伪造数据或来源
- 不直接复用其他 agent 原文，必须二次创作

## 8) 快速调用模式

当用户只说“把这段去 AI 味”，按下面最小输出：
- 诊断（1-2 行）
- 重写稿（1 版）
- 说明改了什么（3 条以内）

当用户说“给我发版稿”，按完整格式输出。
