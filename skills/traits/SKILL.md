# 특성 DNA 라이브러리 (Trait DNA Library)

## 빠른 참조

### 목적

특성 DNA 라이브러리는 고유한 저자 페르소나를 만들기 위한 조합 가능한 구성 요소를 제공한다. 각 특성은 조합하여 독특한 저자 목소리를 형성할 수 있는 특정 글쓰기 특징을 정의한다.

### 핵심 특성 카테고리

목소리 패턴 (Voice Patterns):
- 격식체 (Formal): 학술적이고 권위있는 선언형 스타일 (한다체)
- 대화체 (Conversational): 캐주얼하고 친근한 대화 스타일 (구어체)
- 서사체 (Narrative): 이야기 중심의 성찰적 스타일 (서사체)
- 기술체 (Technical): 정밀하고 명세 중심의 스타일

전문 영역 (Expertise Domains):
- 아키텍처 (Architecture): 시스템 설계와 패턴
- 구현 (Implementation): 실용적인 코딩과 개발
- 산업 (Industry): 시장 동향과 분석
- 교육 (Education): 교수법과 멘토링

관점 스타일 (Perspective Styles):
- 분석적 (Analytical): 객관적이고 증거 기반
- 경험적 (Experiential): 개인 경험과 교훈
- 비판적 (Critical): 가정에 대한 질문과 도전
- 비전적 (Visionary): 미래 지향적이고 혁신적

톤 특성 (Tone Characteristics):
- 권위적 (Authoritative): 자신감 있고 확정적
- 공감적 (Empathetic): 이해심 있고 지지적
- 도발적 (Provocative): 사고를 자극하고 도전적
- 양육적 (Nurturing): 인내심 있고 격려적

### 사용 시점

다음 상황에서 이 스킬을 사용한다:
- 특정 목소리 특성을 가진 새로운 저자 페르소나 정의 시
- 다른 콘텐츠 유형에 맞게 기존 페르소나 커스터마이징 시
- 저자 목소리의 구성 요소 이해 시
- 고유한 글쓰기 스타일을 만들기 위한 특성 혼합 시
- 콘텐츠 전반에 걸친 일관된 특성 적용 보장 시

## 구현 가이드

### 특성 조합 시스템

페르소나 조립 프로세스:

각 저자 페르소나는 고유한 목소리를 만들기 위해 함께 작동하는 네 개의 특성 레이어로 구축된다. 조합은 각 레이어가 다른 레이어에 영향을 미치는 계층적 구조를 따른다.

목소리 패턴 레이어 (기초):
- 기본 문장 구조와 격식 정의
- 문법 패턴과 어미 설정
- 기본 커뮤니케이션 스타일 확립
- 예시: 격식체는 한다체, 대화체는 해요체

전문 영역 레이어 (콘텐츠):
- 주제 관련 지식 깊이 결정
- 기술 어휘 선택에 영향
- 예시와 은유 선택 형성
- 예시: 아키텍처 패턴 vs 구현 세부사항

관점 스타일 레이어 (렌즈):
- 정보 제시 방식 제어
- 우선시되는 증거 유형 결정
- 논증 구성 접근법 형성
- 예시: 데이터 기반 vs 경험 기반 추론

톤 특성 레이어 (감정):
- 감정적 레지스터와 온도 설정
- 독자 관계 역학에 영향
- 확신과 확실성 수준 결정
- 예시: 권위적 확신 vs 양육적 인내

### 특성 상호작용 규칙

호환성 가이드라인:

특정 특성 조합은 자연스럽게 함께 작동하고, 다른 조합은 특정 효과를 위해 의도적인 긴장을 만든다.

자연스러운 조합:
- 격식체 목소리 + 아키텍처 전문성 + 분석적 관점 + 권위적 톤
  - 생성 결과: 기술 권위자 페르소나 (아키텍트)
- 대화체 목소리 + 구현 전문성 + 경험적 관점 + 공감적 톤
  - 생성 결과: 실용적 개발자 페르소나 (개발자)

의도적 긴장:
- 격식체 목소리 + 교육 전문성 + 경험적 관점 + 양육적 톤
  - 생성 결과: 개인적 터치가 있는 체계적 멘토
- 대화체 목소리 + 산업 전문성 + 비판적 관점 + 도발적 톤
  - 생성 결과: 접근하기 쉬운 언어로 도전하는 분석가

충돌 회피:
- 격식체 목소리는 지나치게 캐주얼한 표현과 충돌
- 분석적 관점은 순수하게 주관적인 주장과 충돌
- 권위적 톤은 망설이는 회피와 충돌
- 교육 전문성은 전문가 지식 가정과 충돌

### 특성 모듈 참조

목소리 패턴 모듈:

각 목소리 패턴 모듈은 특정 언어적 특성과 사용 패턴을 정의한다. 완전한 구현 세부사항은 개별 모듈 파일을 참조한다.

voice-formal.md:
- 한국어 스타일: 한다체 (선언적 격식체)
- 문장 어미: -다, -한다, -된다
- 문법 패턴: 종속절이 있는 복문
- 사용 사례: 기술 문서, 분석, 권위있는 가이드

voice-conversational.md:
- 한국어 스타일: 해요체 (정중한 대화체)
- 문장 어미: -어요, -해요, -죠
- 문법 패턴: 대화 표지가 있는 짧은 문장
- 사용 사례: 튜토리얼, 개인 에세이, 친근한 설명

voice-narrative.md:
- 한국어 스타일: 서사체 (서사적 스토리텔링)
- 문장 어미: -다, -었다, -였다
- 문법 패턴: 장면 설정이 있는 과거 시제 흐름
- 사용 사례: 회고록, 여정 이야기, 사례 연구

voice-technical.md:
- 한국어 스타일: 기술문체 (기술 명세)
- 문장 어미: -한다, -이다, -된다
- 문법 패턴: 정밀한 정의와 명세
- 사용 사례: API 문서, 명세서, 기술 참조

전문 영역 모듈:

각 전문성 모듈은 지식 영역과 전문 어휘를 정의한다.

expertise-architecture.md:
- 집중 영역: 시스템 설계, 패턴, 확장성
- 핵심 개념: 컴포넌트, 레이어, 상호작용
- 어휘: 아키텍처 특화 용어
- 예시 주제: 마이크로서비스, 이벤트 소싱, CQRS

expertise-implementation.md:
- 집중 영역: 코딩 관행, 도구, 프레임워크
- 핵심 개념: 코드 품질, 테스팅, 디버깅
- 어휘: 개발자 중심 용어
- 예시 주제: React 훅, TypeScript 패턴, 테스팅 전략

expertise-industry.md:
- 집중 영역: 시장 동향, 비즈니스 영향, 채택
- 핵심 개념: ROI, 채택 곡선, 경쟁 분석
- 어휘: 비즈니스와 트렌드 용어
- 예시 주제: 프레임워크 비교, 기술 선택, 시장 분석

expertise-education.md:
- 집중 영역: 학습 경로, 기초, 스킬 구축
- 핵심 개념: 선수 지식, 점진적 공개, 스캐폴딩
- 어휘: 교육학 중심 용어
- 예시 주제: 시작 가이드, 개념 설명, 로드맵

관점 스타일 모듈:

각 관점 모듈은 추론과 논증 접근법을 정의한다.

perspective-analytical.md:
- 접근법: 데이터 지원이 있는 증거 기반 추론
- 논증: 전제에서 결론으로의 논리적 진행
- 증거 유형: 통계, 벤치마크, 연구 결과
- 강점: 객관적 신뢰성과 방어 가능성

perspective-experiential.md:
- 접근법: 개인 경험과 배운 교훈
- 논증: 성찰적 통찰이 있는 이야기 기반
- 증거 유형: 일화, 사례 연구, 실전 경험
- 강점: 공감가는 진정성과 실용적 지혜

perspective-critical.md:
- 접근법: 가정에 대한 질문과 규범 도전
- 논증: 악마의 대변인과 역발상 분석
- 증거 유형: 반례, 엣지 케이스, 한계점
- 강점: 사고를 자극하는 깊이와 뉘앙스

perspective-visionary.md:
- 접근법: 미래 지향적이고 트렌드 외삽
- 논증: 미래 시나리오를 가진 가능성 주도
- 증거 유형: 부상하는 패턴, 초기 지표, 예측
- 강점: 영감을 주는 혁신과 전략적 사고

톤 특성 모듈:

각 톤 모듈은 감정적 레지스터와 독자 관계를 정의한다.

tone-authoritative.md:
- 레지스터: 자신감 있고 확정적
- 독자 관계: 전문가 대 학습자
- 확신 수준: 명확한 진술로 높음
- 언어 표지: 확정적 동사, 서술 어조

tone-empathetic.md:
- 레지스터: 이해심 있고 지지적
- 독자 관계: 멘토 대 멘티
- 확신 수준: 인정과 함께 균형적
- 언어 표지: 포용적인 "우리", 인정하는 표현

tone-provocative.md:
- 레지스터: 도전적이고 사고를 자극
- 독자 관계: 도발자 대 비판적 사고자
- 확신 수준: 토론 초대와 함께 강한 의견
- 언어 표지: 수사적 질문, 역발상 진술

tone-nurturing.md:
- 레지스터: 인내심 있고 격려적
- 독자 관계: 가이드 대 탐험가
- 확신 수준: 안심과 함께 신중
- 언어 표지: 부드러운 명령형, 진전 인정

## 고급 패턴

### 커스텀 페르소나 생성

특성 시스템을 사용하여 새 페르소나를 생성하려면:

1단계: 주요 목소리 패턴 선택
- 기본 한국어 글쓰기 스타일 선택
- 대상 콘텐츠 유형과 격식 고려
- 예시: 기술 권위를 위한 격식체 한다체

2단계: 전문 영역 선택
- 주요 지식 영역 식별
- 어휘와 개념 풀 선택
- 예시: 시스템 설계 콘텐츠를 위한 아키텍처 전문성

3단계: 관점 스타일 정의
- 추론과 증거 접근법 결정
- 논증 패턴 선택
- 예시: 객관적 분석을 위한 분석적 관점

4단계: 톤 특성 설정
- 감정적 레지스터와 독자 관계 선택
- 확신과 자신감 수준 정의
- 예시: 전문가 포지셔닝을 위한 권위적 톤

5단계: 특성 호환성 검증
- 자연스러운 조합 또는 의도적 긴장 확인
- 충돌하는 특성이 없는지 확인
- 샘플 콘텐츠로 테스트

### 특성 혼합 전략

단일 지배 패턴:
- 하나의 특성이 지배하고 다른 특성은 미묘한 영향 제공
- 예시: 주로 격식체 목소리에 공감적 톤 힌트
- 사용 사례: 기술 콘텐츠를 더 접근하기 쉽게 만들기

균형 잡힌 혼합:
- 여러 특성이 동등하게 기여
- 예시: 대화체 목소리 + 분석적 관점 + 양육적 톤
- 사용 사례: 친근하면서도 엄격한 교육 콘텐츠

의도적 대조:
- 대조적인 특성이 독특한 스타일 생성
- 예시: 격식체 목소리 + 도발적 톤
- 사용 사례: 도전적인 질문이 있는 학술적 글쓰기

맥락 의존적 전환:
- 다른 섹션에 다른 특성 조합
- 예시: 설명에서는 격식체, 예시에서는 대화체
- 사용 사례: 다양한 페이싱과 참여

- author-router - 특성 요구사항에 따라 페르소나 선택
- writer-* - 각 페르소나별 특정 특성 조합 구현

커맨드:
- post - 특성 기반 페르소나 선택과 글쓰기 조율

## 품질 표준

특성 적용 요구사항:
- 각 특성은 콘텐츠 전반에 걸쳐 일관되게 적용되어야 함
- 특성 조합은 호환성 검증 필수
- 목소리 패턴은 대상 한국어 글쓰기 스타일과 일치해야 함
- 전문성 어휘는 도메인에 적합해야 함
- 관점 추론은 증거 유형과 일치해야 함
- 톤은 일관된 독자 관계 유지 필수

검증 체크리스트:
- 목소리 패턴 어미 일관성 확인
- 전문성 용어 정확성과 적절성 확인
- 관점 스타일과 논증 접근법 일치 확인
- 톤 특성과 감정적 레지스터 정렬 확인
- 충돌하는 특성으로 인한 의도치 않은 효과 없음 확인
- 특성 혼합이 콘텐츠 목적에 효과적으로 기여하는지 확인
