# 小红书 GTM 策略

面向跨境电商卖家和科技创业圈的个人IP打造策略。

---

## 核心发布要求 (必须遵守)

### 双账号策略
| 账号 | 定位 | 内容类型 | 语气 |
|------|------|----------|------|
| **个人号** | 创始人/创业者IP | 踩坑故事、个人经验、behind-the-scenes | 口语化、情绪丰富、用"我" |
| **公司号** | 品牌号 | 干货教程、工具使用、行业洞察 | 专业、有数据、用"我们" |

> **关键:** 个人号粉丝粘性是企业号3倍，两个号都要运营

### 发布频率
- **每个账号至少1篇/天**
- 个人号 + 公司号 = **每天至少2篇小红书**
- 配合 LinkedIn (1篇/天) + Twitter (1篇/天) = **每天至少4篇全平台**

### 内容分配原则
| 主题类型 | 个人号 | 公司号 |
|----------|--------|--------|
| 展会/活动 | 我的亲身经历、踩坑 | 行业趋势、参展攻略 |
| 产品相关 | 我为什么做这个 | 功能介绍、使用教程 |
| 行业观察 | 我的观点、吐槽 | 数据洞察、专业分析 |
| 日常更新 | 创业日记、幕后 | 团队动态、客户案例 |

## 内容创作流程 (必须遵循)

每次创作小红书内容时，必须按照以下流程执行：

### Step 1: 研究热门内容

**必做动作:**
1. 搜索小红书相关话题的热门帖子（用WebSearch或浏览器）
2. 记录高赞帖子的：
   - Hook/标题模式
   - 封面设计风格
   - 文案语言特点
   - 互动数据（赞、藏、评）
3. 分析什么让它们成功（情绪点、信息密度、痛点共鸣）

**搜索关键词示例:**
```
小红书 [话题] 热门
小红书 [话题] 爆款笔记
[话题] site:xiaohongshu.com
```

### Step 2: 提取爆款模式

从研究中提取可复用的模式：

| 维度 | 提取内容 |
|------|----------|
| **Hook公式** | 标题的句式结构 |
| **情绪词** | 高频出现的情绪表达 |
| **数字用法** | 如何用数据增强说服力 |
| **视觉风格** | 封面的设计趋势 |
| **互动设计** | 如何引导评论和收藏 |

### Step 3: 用品牌调性改写

**品牌调性:**
- 真诚、硬核、有细节、不鸡汤
- 创业者视角，分享真实经验
- 有数据支撑，不说空话

**改写原则:**
1. 保留爆款的情绪钩子和结构
2. 替换为自己的真实案例和数据
3. 用口语化表达，避免书面语
4. 加入具体数字（$400→$180, 省$1000+）
5. 用情绪词增强共鸣（亏麻了、稳了、社死）

### Step 4: 输出完整内容

**交付物清单:**
- [ ] 标题（含SEO关键词）
- [ ] 正文（口语化、有情绪、有数据）
- [ ] 6张图片的图片生成Prompts（2K分辨率: 2160×2880px）
- [ ] Hashtags
- [ ] 发布时间建议
- [ ] 评论区补充内容
- [ ] 互动回复模板

| 行为 | 权重 | 含义 |
|------|------|------|

| 评论/转发 | 各4分 | 深度互动胜于被动浏览 |
| 点赞/收藏 | 各1分 | 基础信号 |

- **初始流量池:** 100-500曝光
- **2小时内**互动率决定是否进入下一级流量池

- **70%用户使用搜索功能** — 标题SEO至关重要

## 内容框架

### 四种高转化内容类型

1. **创业故事型**
   - "我，30岁，辞职做AI产品，第一个月发生了什么"

2. **干货教程型**
   - "跨境电商人必看｜用这个AI工具，Listing效率翻3倍"

3. **痛点解决型**
   - "亚马逊广告烧钱没效果？这个方法让我ACoS降了40%"

4. **工具评测型**
   - "测评10个AI Listing工具，这3个值得花钱"

### 爆款标题公式

```
目标人群 + 痛点/场景 + 数字化解决方案 + 情绪词
```

**模板:**
- "跨境卖家救命｜这5个AI工具让我多睡3小时"
- "后悔没早知道！80%的跨境人都不会用的AI功能"
- "说一个很变态但解决80%问题的方法"

## 视觉规范

### 分辨率要求 (重要)

| 类型 | 比例 | 最低分辨率 | 推荐分辨率 (2K+) |
|------|------|-----------|-----------------|
| **竖版封面** | 3:4 | 1080×1440px | **2160×2880px** |
| **正方形** | 1:1 | 1080×1080px | **2160×2160px** |
| **横版** | 4:3 | 1440×1080px | **2880×2160px** |

> **注意:** 生成图片时，必须指定2K分辨率（2160×2880px for 3:4），确保画质清晰。小红书会压缩图片，高分辨率原图能保证压缩后仍然清晰。

### 封面类型
- 文字型、对比型、多图拼贴、场景氛围、人物出镜
- **统一系统:** 品牌色+固定字体+模板复用

### 高点击率封面设计趋势 (2024-2025)

#### 核心原则: 3秒法则
- 视觉冲击力 > 内容深度
- 先抓眼球，让用户愿意点开

#### 6大爆款套路

| 套路 | 描述 | 适用场景 |
|------|------|---------|
| **质感为王** | 高清+氛围感调色 | 所有内容 |
| **标题党排版** | 痛点+解决方案，大字突出关键词 | 干货教程 |
| **强反差对比** | Before/After、数字对比 | 对比测评 |
| **图文结合** | 纯色底+钩子文案 | 观点类 |
| **多图统一风格** | 合集测评必备 | 清单、评测 |
| **创意脑洞** | 个人风格+热点结合 | 创业故事 |

#### 高信息密度设计

**设计要点:**
- 每张图展示3-4个信息点，让用户感到"内容丰富"
- 用数字/数据强化说服力（"省50%"、"8个meeting"）
- 关键词放大，细节缩小，形成视觉层级
- 卡片式布局增加信息密度感

**信息钩子技巧:**
- 展示"信息量大"，让人想点开看完整内容
- 部分信息需要点开才能看清（知识性好奇心）
- 清单/Checklist类内容收藏率最高

#### 真人出镜效果

- 真人照片提升信任感和亲和力
- CTR提升30%+（相比纯文字/纯图）
- **推荐:** 封面留真人照片位 + 文字信息组合

#### 封面四大特征

1. **主体突出** — 人物或产品处于视觉焦点
2. **信息明确** — 一眼看懂内容主题
3. **审美在线** — 符合小红书精致调性
4. **适当"钩子"** — 引发好奇或共鸣

#### 需要避免的设计

- 低质量现拍照片、随意下载的网图
- 背景杂乱或模糊不清
- 信息太少，显得空洞
- 纯品牌宣传，无个人视角

## 封面文案语言模式 (重要)

小红书封面上的文字是"第二个标题"，直接影响点击率。以下是爆款文案的语言逻辑：

### 核心原则：口语化

> "口语化是基础，一定要转化成用户听得懂的话语"

| 避免 | 推荐 |
|------|------|
| "酒店预订延迟导致成本增加" | "酒店订晚了，多花了$200" |
| "会议安排不当" | "到现场才发现没人理我" |
| "演示效果不佳" | "Demo黑屏，场面一度很尴尬" |
| "提前规划的重要性" | "早3个月准备，省一半钱" |

### 文案公式

#### 公式1: 痛点 + 解决方案 + 结果
```
"酒店订晚了$400/晚 → 9月订只要$200"
"0个meeting → 提前约，确认8个"
```

#### 公式2: 事件 + 情绪词
```
"Demo黑屏，我当场社死"
"酒店满了，我人傻了"
"8个meeting确认，稳了"
```

#### 公式3: 细分人群 + 痛点 + 解决方案
```
"创业者参展必看｜这3个坑我替你踩了"
"AI创业者避坑｜提前3个月做这些"
```

#### 公式4: 数字对比
```
"去年$400/晚 → 今年$180/晚"
"0个meeting → 8个confirmed"
"准备3个月 vs 准备3天，差距太大"
```

### 爆款情绪词库

**正面情绪（成功/收获）:**
- 稳了、搞定、确认、省下、翻倍
- 太香了、绝了、赢麻了

**负面情绪（踩坑/教训）:**
- 傻了、懵了、社死、尴尬、亏大了
- 人麻了、血亏、踩坑

**号召/互动:**
- 建议收藏、照着做、别像我一样
- 划重点、家人们、真心建议

**强调词:**
- 吐血整理、纯干货、亲测有效
- 手把手、保姆级、一文讲透

### 封面文案模板

**对比型（Before/After）:**
```
去年：[痛点描述] ❌
今年：[解决方案] ✓
```

**清单型:**
```
展会准备清单
□ 展前3个月：订酒店、列名单
□ 展前1个月：约meeting、备Demo
□ 展前1周：测试3遍
```

**数据型:**
```
这次准备了：
✓ 9月订酒店，省$1000+
✓ 8个meeting已确认
✓ 离线Demo，0故障
```

**观点型:**
```
"展会不是去看热闹的"
"是去找机会的"
— 准备越充分，收获越大
```

### 文案长度建议

| 位置 | 字数 | 说明 |
|------|------|------|
| 主标题 | 4-8字 | 核心信息，最大字号 |
| 副标题 | 8-15字 | 补充说明或情绪词 |
| 要点列表 | 每条≤15字 | 简洁有力，可扫读 |
| 底部CTA | ≤10字 | 引导互动或收藏 |

### 避免的表达

- ❌ 书面语、官方腔（"关于XX的重要提示"）
- ❌ 过于正式（"本次分享将为您介绍..."）
- ❌ 空洞形容词（"很好"、"非常重要"）
- ❌ 没有具体数字（"省了很多钱" → "省了$1000"）

## 展会/活动类内容策略 (CES/MWC等)

### 高赞模式分析

**头部账号案例: 展会笔记爆款**
- 高赞: "准备，不是12月才开始的" (时间线干货)
- 高赞: "去CES前没人会当面告诉你的那些事儿" (内幕避坑)
- 中赞: "很多展位其实只是在消耗时间" (观点吐槽)

**低赞反例: 品牌官方号**
- 同标题发十几篇，每篇0-1赞
- 问题: 纯宣传、无价值、无个人视角

### 高赞Hook公式

| 类型 | 模板 | 效果 |
|------|------|------|
| 打破常识 | "XX月开始准备是来不及的" | 制造认知冲突 |
| 揭秘内幕 | "没人会当面告诉你的那些事" | 稀缺信息感 |
| 观点锐利 | "很多展位只是在消耗时间" | 引发讨论 |
| 避坑指南 | "第一次踩的雷" | 实用价值 |

### 内容结构模板

```
1. Hook (打破常识/揭秘内幕)
2. 分阶段/分步骤 (有具体数字)
   - 时间线: 10月/11月/12月/1月
   - 场景: 进场 → 开展 → 撤展
   - 清单: 20个关键动作、30-20-10天节点
3. 具体案例 (踩过的坑、花过的冤枉钱)
4. 金句收尾
   - "希望你是带着清单进场，而不是带着账单离场"
```

### 定位策略

| 避免 | 推荐 |
|------|------|
| "第一次去XX展" | "二刷XX展" / "第N次参展" |
| 小白视角 | 老手分享 |
| 记录流水账 | 观点+干货 |
| 纯品牌宣传 | 个人视角+真实经验 |
