# 看房作战方法论

## 看房前准备

### 人设选择矩阵
| 人设 | 适用条件 | 优势 | 风险 | 推荐指数 |
|------|----------|------|------|----------|
| 投资客 | 不急、价格敏感 | 随时可走、谈判主动 | 中介可能不重视 | ⭐⭐⭐⭐ |
| 自住不急 | 有明确需求、时间充裕 | 中介重视且你不被动 | 需要演得可信 | ⭐⭐⭐⭐⭐ |
| 帮人看房 | 需要隔离身份 | 套情报方便、零压力 | 决策链长 | ⭐⭐⭐ |
| 刚需急买 | 确实急需且已锁定目标 | 中介全力推进 | 砍价空间被压缩 | ⭐⭐ |

**建议**：首次看房用"投资客"或"自住不急"（保持主动权），确定要买后切到"自住但时间灵活"。

### 看房路线规划
- 看多套房：**先看非目标房**（找手感、收集信息、建立对比基准）
- **最后看目标房**（此时你已有市场感觉，经纪人也已"热身"）
- 同小区看多套时：避免让经纪人发现你的真正目标

### 实时协作设定
与 AI 约定看房时的沟通方式：
- 手机随手发消息：经纪人说了什么、房子什么状况、你的感受
- AI 后方做什么：实时交叉比对前期数据、标注经纪人言语中的关键信息
- 重点捕捉：**经纪人无意中泄露的锚点价、历史出价、业主情况**

## 实地 Checklist

### 🏠 房屋硬件（必检）
- [ ] **采光**：各房间自然光情况（最好分上午/下午看）
- [ ] **朝向**：主卧客厅朝向 + 前方有无遮挡建筑
- [ ] **层高**：实际感受（老房标 2.8 实际可能 2.6）
- [ ] **墙面**：裂缝、渗水痕迹、发霉、起泡、脱落
- [ ] **天花板**：渗水印、裂缝（尤其顶楼）
- [ ] **地面**：地板质量、瓷砖空鼓（指节敲）
- [ ] **门窗**：密封性、隔音效果、是否正常开关
- [ ] **水压**：厨卫水龙头全开、热水器响应速度
- [ ] **下水**：厕所冲水是否顺畅、厨房下水堵塞情况
- [ ] **电路**：开关插座数量位置、有无老化迹象
- [ ] **管线**：暖气位置、煤气表、天然气接入情况

### 🏢 楼栋与公共区域
- [ ] **楼道**：整洁度、照明、消防设施
- [ ] **电梯**（如有）：数量、品牌、运行状态
- [ ] **地下室/储藏室**：是否包含、面积
- [ ] **顶楼**：有无违建加盖
- [ ] **快递/信报**：有无代收设施

### 🌳 小区环境
- [ ] **物业维护**：公共区域状态、门禁、监控
- [ ] **绿化**：实际绿化率
- [ ] **停车**：地面/地下、固定/临时、月租费
- [ ] **噪音源**：周边道路、学校、工厂、工地
- [ ] **异味源**：餐饮排烟、垃圾站距离
- [ ] **邻居**：楼道观感、有无群租迹象

### 📱 生活便利
- [ ] **手机信号**：各运营商室内信号（老楼低层常有盲区）
- [ ] **宽带**：光纤是否入户
- [ ] **周边配套**：超市、医院、地铁、学校距离

## 套情报方法论

### 引出经纪人锚点价
| ❌ 不要问 | ✅ 应该问 | 为什么 |
|-----------|-----------|--------|
| "你觉得值多少？" | "如果成交的话，首付大概怎么算？" | 他会用一个价格做假设 → 暴露锚点 |
| "最低多少？" | "之前出价的人大概什么水平？" | 引出历史数据而非主观判断 |

### 了解历史看房
- "之前看的人多吗？有谈过没谈拢的吗？大概差多少？"
- 不要问"有人出过价吗"（他可能编一个高价抬你的锚点）

### 了解业主动机
- "业主是搬去哪？换房还是..."（判断紧迫度）
- "这套挂了多久了？中间调过价没？"（判断底线走势）
- "业主本人在吗？还是家人在打理？"（判断是否多人共有 → 找分歧）

### 制造竞品压力
- "我上午看了 XX 小区的一套，同面积但便宜不少"
- "你们楼 X 层那套也在卖？好像更划算"
- 不需要是真的——但如果你在 Phase 2 做了竞品分析，那就是真的

## 看房后——信息更新流程

看完房立即做：
1. **记录**：用看房记录卡模板逐项填写
2. **对比**：新信息 vs 前期数据，有没有矛盾或新发现？
3. **更新弹药**：经纪人透露的新信息是否产生了新弹药？
4. **调整策略**：业主底线的估计是否需要修正？
