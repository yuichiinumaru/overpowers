# 房源情报网络构建方法论

## 数据来源评估

### 线上平台
| 平台 | 数据特点 | 适用场景 |
|------|----------|----------|
| 贝壳/链家 | 数据最全、成交记录可信度高、有 VR 看房 | 主要信息源 |
| 安居客 | 房源覆盖广、部分独家房源 | 补充搜索 |
| 我爱我家 | 自有门店房源 | 特定区域补充 |
| 幸福里/诸葛找房 | 成交数据分析好、有走势图 | 趋势分析 |
| 中国房价行情网 | 宏观统计数据 | 大盘判断 |

### 线下渠道（用户配合）
- 物业/门卫：知道谁在卖但没挂网
- 小区业主群：有些房源只在群里发
- 片区中介门店：走进去问，比网上信息更及时

## 三层数据采集方法

### Layer 1：目标小区全量房源
```
web_search: "{小区名} 在售二手房"
web_search: "{小区名} 贝壳找房"
web_search: "{小区名} 链家在售"
```
用 browser 打开贝壳/链家页面，逐套采集：
- 每套的链接、价格、面积、户型、楼层、朝向
- 截图保存（房源可能随时下架或调价）

### Layer 2：历史成交数据
```
web_search: "{小区名} 成交记录 {当前年份}"
web_search: "{小区名} 历史成交价"
```
重点字段：
- 同户型近 6 个月成交价（注意区分成交价 vs 挂牌价）
- 同楼栋不同楼层的历史成交价差
- 成交周期（从挂牌到成交用了多久）

### Layer 3：周边竞品扫描
```
web_search: "{区域名} 二手房 {价格区间} {面积区间}"
```
找出 1km 内的替代方案：
- 作为"我也在看别的"的底气
- 如果竞品更便宜 → 直接拿来压价

## 关键分析维度

### 挂牌时长分析
| 时长 | 信号 | 策略含义 |
|------|------|----------|
| < 1 月 | 新挂，价格可能虚高 | 观察，不急出手 |
| 1-3 月 | 正常周期 | 常规砍价 |
| 3-6 月 | 业主在松动 | 重点砍 |
| > 6 月 | 价格严重偏高或有问题 | 大胆砍或放弃 |

### 调价轨迹分析
- 降过 1 次 → 业主心理防线已打开
- 降过 2+ 次 → 底线在持续下移，每次降多少反映紧迫度
- 涨过价 → 可能是中介建议"先挂高再降"的套路
- 长期不调 → 不急，或不是真想卖

### 同户型价差分析（最重要）
同楼不同楼层的同户型在售 → 这是砍价的核心武器：
- 无电梯楼：低楼层（2-3 层）反而是黄金楼层，高楼层应该更便宜
- 有电梯楼：高楼层通常溢价，但要看遮挡和噪音
- 同户型价差 = 你砍价的"铁证"

### 空置状态评估
- 房源已空置 → 每月持有成本 = 物业费 + 贷款利息 + 租金机会成本
- 空置越久，卖方的"出手意愿"越强
- 这是心理弹药的重要来源

## 平台使用技巧

### 贝壳/链家特有信息
- 页面**不显示楼栋号**（只到小区级别）→ 需要问经纪人确认
- **VR 看房**有时能看到楼栋平面图
- **经纪人聊天**中的无意透露比正式报价更值钱
- 经纪人用某个价格做假设时（"如果成交价 XX"）→ 他暴露了心理锚点

### 多经纪人策略
- 同一套房可以有多个经纪人 → 选合作最顺畅的
- 选择标准：回复速度 > 从业年限 > 专业证书 > 门店大小
- 店经理通常比普通经纪人有更大的谈判空间和资源
- 不要因为人情不好意思换经纪人——这是纯商业关系

### 隐私保护
- 不要在平台聊天里说"我住隔壁""必须买""预算 XX"
- 经纪人之间可能共享客户信息
- 保持信息模糊性 = 保持谈判筹码
