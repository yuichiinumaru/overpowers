# 小红书内容模板框架（可扩展）

## 0. 目标

为不同内容类型提供统一模板路由机制，减少临时判断成本。模板用于“组织给定内容”，不用于重新生成事实内容。

## 1. 路由规则

1. 若输入提供 `template_type`，优先按该模板执行。
2. 若未提供，则按素材特征匹配：
   - 含论文标题/作者/章节图等结构化论文信息 -> `paper-interpretation`
   - 否则回退到通用模板（直接填充）。
3. 模板缺失时，回退通用模板并明确提示。

## 2. 通用模板（direct-fill）

适用：普通图文、长文、视频的大多数直发场景。

- 图文（`post_type=image`）映射：
  - `image_text` -> 图片编辑文案区（可空）
  - `title` -> 标题区
  - `body` -> 正文区
- 长文/视频映射：
  - 维持各自默认表单映射，不强制图文三段式。
- 封面与素材：优先使用用户提供首图/视频。
- 不重写、不扩写、不改写事实。

## 3. 论文解读模板（paper-interpretation）

适用：论文解读、论文速读、学术内容拆解。

### 3.1 封面规则
- 首图包含文章名和作者信息。
- 若已有现成封面，优先使用；否则在首图文案层展示文章名+作者。

### 3.2 配图规则
- 将文章中所有的图都作为配图
- 若超过十张图，只选取前十张图并提示超额。

### 3.3 文案规则
- 图文发布时，必须映射为 `image_text/title/body` 三段后再写入对应区域。
- 长文/视频发布时，沿用各自默认映射流程。
- 标题与正文直接使用给定内容。
- 不补写论文结论，不新增未提供实验结果。
- 信息不足时明确标注“原始素材未提供”。

## 4. 扩展新模板规范

新增模板时，至少补充以下四项：
1. 触发条件（输入特征或 `template_type`）
2. 封面规则
3. 素材映射规则（图片/视频/正文）
4. 降级策略（素材不全时如何处理）
