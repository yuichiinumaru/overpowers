## API統合

### YouTube Data API

```javascript
// YouTube アップロード
const youtube = google.youtube({
  version: 'v3',
  auth: oauth2Client
});

const uploadVideo = async (videoPath, metadata) => {
  const res = await youtube.videos.insert({
    part: ['snippet', 'status'],
    requestBody: {
      snippet: {
        title: metadata.title,
        description: metadata.description,
        tags: metadata.tags,
        categoryId: metadata.categoryId
      },
      status: {
        privacyStatus: 'private', // 後で公開
        publishAt: metadata.publishAt,
        selfDeclaredMadeForKids: false
      }
    },
    media: {
      body: fs.createReadStream(videoPath)
    }
  });

  return res.data;
};
```

### サムネイル設定

```javascript
// サムネイルアップロード
const setThumbnail = async (videoId, thumbnailPath) => {
  await youtube.thumbnails.set({
    videoId,
    media: {
      body: fs.createReadStream(thumbnailPath)
    }
  });
};
```

## 分析・最適化

### パフォーマンス追跡

```yaml
Key Metrics:
  - Views（視聴回数）
  - Watch Time（総再生時間）
  - CTR（クリック率）: 目標 5%+
  - Average View Duration（平均視聴時間）
  - Subscriber Growth（登録者増加）

Weekly Analysis:
  1. 各動画のパフォーマンス確認
  2. 成功/失敗パターン分析
  3. サムネイル A/Bテスト結果
  4. 視聴者維持グラフ分析
  5. 次週の戦略調整
```

### A/Bテスト

```yaml
Test Elements:
  - サムネイル: 2-3パターン
  - タイトル: キーワード位置、表現
  - 公開時間: 時間帯テスト
  - 動画の長さ: 最適長さ探索

Methodology:
  - 1つの要素のみ変更
  - 十分なサンプルサイズ
  - 統計的有意性確認
  - 成功パターンを記録
```

  analysis:
    1. サムネイル問題？

    3. SEO問題？
       - 表示回数低い → キーワード再選定
    4. タイミング問題？

    - 失敗パターンを記録

    - 次回の企画に反映

  storage:
    path: /moltworker/data/youtube-learning.json
    metrics:
      - success_patterns
      - failure_patterns
      - keyword_performance
