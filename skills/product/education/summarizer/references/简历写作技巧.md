# 简历写作技巧大全

## 一、简历核心原则

### 一页纸原则
- **校招/实习**：严格一页纸
- **社招3年以内**：尽量一页纸
- **社招3年以上**：可适当增加，但不超过2页

### 倒序原则
- 最近的经历放最前面
- 每个模块内时间倒序

### 相关性原则
- 只写与目标岗位相关的经历
- 不相关的经历可以简写或不写

---

## 二、STAR法则详解

### 什么是STAR
- **S (Situation)**：背景，为什么要做这个项目
- **T (Task)**：任务，你的职责和目标
- **A (Action)**：行动，具体做了什么
- **R (Result)**：结果，取得了什么成果

### 应用示例

**❌ 原写法**
负责用户增长策略，通过活动策划提升了用户量。

**✅ STAR优化**
- **背景**：新功能上线初期用户增长缓慢，DAU仅X万
- **任务**：作为增长owner，目标3个月内DAU提升至X万
- **行动**：
  - 拆解增长漏斗，识别关键转化节点
  - 设计裂变激励机制，降低参与门槛
  - 协调市场、技术、设计3个团队推进
- **结果**：DAU提升至X万（+X%），超预期X%

## 三、数据化表达技巧

### 数据前置原则
每段经历的第一句话放核心数据。

```
❌ 负责XX项目，通过XX方法，最终用户增长X万
✅ 用户增长X万（+X%）| 负责XX项目，通过XX方法实现增长突破
```

### 常用数据维度

| 岗位 | 核心数据 |
|:---|:---|
| 产品经理 | DAU、转化率、留存率、功能使用率、项目上线率 |
| 产品运营 | 新增用户、留存率、GMV、活动参与率、ROI |
| 数据分析 | 分析项目数、指标提升幅度、模型准确率 |
| 商业化 | 收入、ARPU、CPM、广告主数、变现效率 |
| 策略产品 | 点击率、转化率、策略覆盖率、实验效果 |

### 数据表达技巧

**绝对值 vs 相对值**
```
- 绝对值：用户增长10万
- 相对值：用户增长10万（+20%）
- 对比值：用户增长10万（高于大盘15%）
```

**没有精确数据时的表达**
```
- 估算：约X万、超X万、近X万
- 范围：X-X万、数十量级
- 定性：显著提升、大幅提升、用户好评
- 对比：从第X名提升至第X名、超过竞品
```

## 四、关键词植入技巧

### 为什么要植入关键词
1. **ATS系统**：自动筛选简历时会匹配关键词
2. **HR快速扫读**：10秒内找关键词判断匹配度
3. **面试官引导**：方便面试官问相关问题

### 各岗位关键词清单

**产品经理**
- 用户研究、需求分析、PRD、原型设计
- A/B测试、数据分析、数据驱动
- 跨部门协作、项目管理、owner意识
- 增长、留存、转化率

**产品运营**
- 用户增长、拉新、留存、促活
- 活动策划、裂变、GMV、DAU
- 用户分层、社群运营、内容运营
- 数据分析、ROI、转化率

**数据分析**
- SQL、Python、统计学、机器学习
- 指标体系、漏斗分析、归因分析
- A/B测试、假设检验、显著性
- 数据可视化、数据看板

**商业化产品**
- 广告、CPM、CPC、eCPM、ROI
- 会员、订阅、LTV、ARPU
- 变现、收入、商业化率
- 定价策略、竞价策略

**策略产品**
- 推荐策略、搜索策略、召回、排序
- 特征工程、模型评估、A/B测试
- AI、大模型、Prompt工程
- 算法、机器学习、深度学习

## 五、常见错误与修正

### 错误1：描述过于笼统
```
❌ 负责用户增长工作，取得了很好的效果
✅ 负责用户增长工作，3个月内DAU从X万提升至X万（+X%）
```

### 错误2：强调"参与"弱化贡献
```
❌ 参与XX项目，协助完成XX功能
✅ 负责XX模块，独立完成PRD撰写和项目推进，上线后DAU提升X%
```

### 错误3：堆砌技术名词
```
❌ 使用React、Vue、Node.js、MySQL、Redis开发系统
✅ 负责XX系统后端开发，优化数据库查询性能，响应时间降低X%
```

### 错误4：格式不统一
```
❌ 时间格式混用（2024.06 / 2024/6 / June 2024）
✅ 统一格式：2024.06 - 2024.09
```

### 错误5：错别字和语病
- 务必检查错别字
- 注意"的/地/得"用法
- 保持主谓一致

## 六、不同岗位的表达侧重

### 产品经理 - 突出"决策"和"推动"
```
- 决策：基于用户调研决定[XX方向]
- 设计：输出PRD、原型，设计[XX功能]
- 推动：协调[XX团队]，推动项目按期上线
- 验证：通过A/B测试验证，[XX指标]提升X%
```

### 产品运营 - 突出"策略"和"数据"
```
- 策略：设计[XX]增长策略，覆盖[XX渠道]
- 执行：落地[XX]活动，协调[XX资源]
- 数据：监控[XX指标]，根据数据优化[XX]
- 成果：新增用户X万，ROI达X
```

### 数据分析 - 突出"分析"和"洞察"
```
- 问题：发现[XX指标]异常，定位[XX问题]
- 分析：通过[XX方法]分析，发现[XX规律]
- 建议：提出[XX建议]，预期[XX效果]
- 落地：推动优化，实际[XX指标]提升X%
```

## 八、简历迭代建议

### 第一次迭代：内容完整
- 把所有经历写上去
- 不纠结措辞和数据

### 第二次迭代：STAR优化
- 每个项目按STAR法则重写
- 补充数据和细节

### 第三次迭代：精简优化
- 删减不相关内容
- 压缩到一页纸
- 优化措辞

### 第四次迭代：岗位定制
- 根据具体JD调整关键词
- 调整项目排序
- 强调相关经历

### 寻求反馈
- 给学长/朋友看
- 给目标公司的前辈看
- 根据反馈迭代

1. 内推（成功率最高）

- 被拒后可询问原因以便改进

## 十、常见问题解答

**Q1：没有实习经历怎么写？**
- 写课程项目、社团经历、竞赛经历
- 突出项目管理和数据分析能力
- 个人账号运营（小红书/抖音/B站）

**Q2：实习时间很短怎么写？**
- 强调成果而非时间
- 突出独立负责的部分
- 如果<3个月，可合并到一段经历

**Q3：转行怎么写简历？**
- 突出 transferable skills
- 补充目标岗位的相关项目
- 解释转行动机

**Q4：GAP year怎么解释？**
- 诚实说明原因（考研/创业/休息）
- 强调期间的学习/成长
- 不要隐瞒，会被问到

**Q5：可以包装经历吗？**
- 可以优化措辞，但不要造假
- 夸大成果会被追问细节
- 诚信是底线
