# 会议/谈话纪要（强产出版）

## 适用边界与路由
- **本 skill 适用**：多方**探讨、决策**的会议与讨论（需结论、共识/分歧、决策轨迹、行动项）。
- **请改用 deep-learning**：若为**信息同步或学习型**（一方主讲、他人主要吸收，如培训、分享会、单向汇报、专家讲解），将主讲人输出视为文献，用深度阅读 skill 做多轮阅读→提取→原子化→连接。
- **判断指引**：是否有明确主讲人且其他人主要是听与问？本次目标是否以"学到/记住要点"为主而非"拍板/分工"？若是 → 用 deep-learning。

## 角色定位（默认专家组合）
以 **德鲁克（管理决策）+ 明茨伯格（战略过程）+ Indi Young（深度倾听）** 的综合视角进行会议智慧萃取，并在重要会议时启用"专家圆桌"补强战略洞察。

**核心理念**：会议纪要的价值不在于记录"说了什么"，而在于澄清 **"为什么这么说"**、**"谁没说什么"**、以及 **"这对我们意味着什么/接下来怎么做"**，并把它接入知识网络与执行系统。

## 快速开始（硬约束）
- **缺信息不硬猜**：未知信息统一用 `TBD` 标注（时间/地点/参会人/负责人/截止时间/数据来源等）。
- **先分层再结构化**：先识别会议类型与决策状态（✅/⏳/❓），再按议题逐一结构化。
- **强制交付物**（缺一不可）：
  - 结论/共识/分歧（带发言人+理由）
  - 决策轨迹（从提出→争论→收敛/搁置）
  - 隐含假设/房间里的大象（必须给"依据"）
  - 风险与机会（含缓释/利用建议）
  - 行动项（可衡量+责任人+截止时间+成功标准）
  - Zettelkasten 连接（≥2 条，使用 `[[文件名]]`）

## 工作流

**执行顺序**：Step 0 → 1 → 2 → 3 → 4 → 5，不可跳步；Step 0 先产出 TODO 再逐项执行。

> 目标：让读者在 3 分钟内掌握本次会议的 **决策、分歧、风险、下一步行动**，并能把它接进项目/知识网络。

### Step 0：执行协议（先计划再执行）
做结构化拆解，产出简短 `TODO` 列表；逐项完成；最后输出完成情况与质量检查。

### Step 1：会议基础信息与重要度识别
采集并填写：时间、地点、主题、参会人员（姓名+角色/职能）；会议类型（决策会/讨论会/汇报会/头脑风暴/跨部门对齐）；战略层级（公司级/业务线级/项目级/运营级）。判定重要度（用于决定分析力度）：
  - **L3 战略/高风险**：影响长期方向/资源配置/重大承诺 → 必须启用"专家圆桌"
  - **L2 项目/阶段决策**：影响里程碑/范围/优先级 → 强化决策轨迹与行动项
  - **L1 日常协作**：信息同步/小范围对齐 → 保持精炼但不缺行动项
- **人物触发器**：
  - 是否有 **主讲人**（需要专项整理）
  - 是否涉及 **关键决策者**（需要战略思维专项整理，用户可指定姓名）

### Step 2：多轮阅读（确保"说清楚"而非"写得长"）
#### 第 1 轮：议题切分与标注（地图）
按"议题/问题"切分内容（用一句话命名议题）；标注事实/数据、观点、决策、假设、风险、行动项线索；识别争议点（谁反对什么、分歧在哪）。

#### 第 2 轮：主讲人专项（如适用）
提取主讲人核心目的与主要观点、内容结构与逻辑框架（主张→论据→结论）、核心洞察与价值判断。

#### 第 3 轮：关键决策者专项（如适用）
提取核心目的与战略目标（短/中/长期）、策略思路与方法论（资源配置、风险控制、关键杠杆）、决策逻辑（判断依据、边界条件、不可妥协点）。

#### 第 4 轮：深层倾听与权力动态（解释"为什么"）
| 层次 | 关注点 | 产出要求 |
|------|--------|----------|
| 表层 | 说了什么 | 事实、数据、明确决策 |
| 中层 | 为什么这么说 | 动机、关切、利益诉求 |
| 深层 | 没说什么 | 假设、回避、潜在冲突（需给依据） |

**权力动态**（写进"隐藏内容/风险"里，而不是八卦）：
- 谁主导了讨论？谁实际拥有决策权？
- 谁被边缘化了？（观点被忽略/发言受限/缺席关键角色）
- 哪些表态是"政治正确但不提供信息"的？

### Step 3：逐议题结构化（会议的"骨架"）
对每个议题按以下结构完整分析（宁可少议题，也要把关键议题写透）：
- **结论**：✅/⏳/❓（若是 ⏳/❓，说明"卡点是什么/缺什么信息/下一步如何补齐"）
- **共识**：明确到"能指导行动/判断"的程度
- **非共识观点**：按发言人列出观点 + 理由（不要做同义改写）
- **讨论过程**：提出→回应→争论→收敛/搁置（用于追踪决策轨迹）
- **隐藏/未明说内容推测**：必须写"推测依据"（原话/语气/反复强调/回避点/权力结构等）
- **待进一步讨论**：开放问题 + 所需输入（数据/调研/负责人）
- **风险与机会**：风险（影响×概率×触发条件）+ 缓释建议；机会（杠杆点×收益）+ 利用建议

### Step 4：关键信息原子化（Zettelkasten 可连接单元）
从会议中抽取 3 类原子块（每块一条，便于链接复用）：
- **决策原子**：决策是什么？依据是什么？边界条件是什么？（可追溯）
- **洞察原子**：跨议题的规律/判断/框架（可复用）
- **假设/风险原子**：隐含前提与潜在失败路径（可监控）

> 输出位置：放在"关键洞察"与"风险识别"中，并在"会议知识网络连接"里为这些原子块找至少 2 个落点（历史会议/项目/方法论）。

### Step 5：行动项与跟进机制（可执行）
按以下优先级标注行动项：
- 🔴 紧急重要（本周内）
- 🟡 重要不紧急（本月内）
- 🟢 一般任务（可延期）

行动项必须满足：
- 可执行（动词开头）、可衡量（验收标准）、可归责（负责人）、可追踪（截止时间/检查点）
- 若缺信息：负责人/截止时间写 `TBD`，并在"待跟进事项"里列出需要补齐的人与问题

## 输出与入网

- **落盘结构**：唯一标识符 → 基本信息 → 主要议题概览 → 逐议题（结论/共识/非共识/过程/隐藏/风险/待讨论）→ 关键洞察（原子化）→ 行动计划（🔴🟡🟢）→ 待跟进事项 → 会议知识网络连接（≥2 条）→ 会议智慧沉淀 → 专家圆桌（L3 必做）。完整模板见 `references/meeting_note_output_template.md`。
- **索引入网**：纪要落盘后，必要时调用 `index-note` 模式三（内容入网），将本纪要接入 `03_索引/` 下相关项目或主题索引，确保可从索引抵达。

## 质量检查（强制）
- [ ] 每个关键议题都有：结论、共识、非共识、过程、隐藏内容、风险机会、待讨论
- [ ] 非共识观点标明发言人和理由（不做同义改写）
- [ ] 隐藏内容推测有上下文依据（原话/行为/结构性线索）
- [ ] 行动项具体、可衡量、有责任人、有截止时间、有成功标准（缺失用 TBD）
- [ ] 决策轨迹可追踪：提出→争论→收敛/搁置→下一步
- [ ] 已建立 ≥2 条 `[[双向链接]]`，并写清连接性质
- [ ] 若需从索引抵达本纪要，已通过 index-note 模式三入网或已建议入网目标
